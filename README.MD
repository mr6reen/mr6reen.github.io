PROJECT NAME
===============================================================================
Caloriq — PWA calorie tracker with user-defined meal presets, for people
who want dead-simple food logging without giant food databases.
URL: https://kcalabs.com (домен в планах, пока не куплен)
===============================================================================

STACK
• Frontend: Vanilla JS, Tailwind CSS (CDN), HTML5
• Backend: нет (MVP — всё на localStorage, бэк не нужен)
• Database: localStorage (браузер) — данные хранятся локально на устройстве
• Fonts: Anton + Roboto (Google Fonts CDN)
• PWA: manifest.json + service worker (sw.js) для офлайн работы и установки


HOW TO RUN LOCALLY
• Скопировать все файлы из public_html/ в любую папку
• Запустить локальный сервер: php -S localhost:8080
• Открыть http://localhost:8080/onboarding-1.html
• Для теста на телефоне: php -S 0.0.0.0:8080, открыть http://[IP компа]:8080/onboarding-1.html
• Телефон и комп должны быть в одной WiFi сети


FILE STRUCTURE
public_html/
│
│   # ── Экраны приложения ──────────────────────────────────────────────────
│
├── onboarding-1.html
│     Шаг 1 онбординга. Юзер выбирает цель: похудеть / держать / набрать вес.
│     Сохраняет 'goal' в localStorage. Переходит на onboarding-2.html.
│
├── onboarding-2.html
│     Шаг 2 онбординга. Юзер вводит дневную цель калорий вручную или через
│     калькулятор (формула Mifflin-St Jeor). Учитывает цель из шага 1.
│     Сохраняет 'calories' в localStorage. Переходит на index.html.
│
├── index.html
│     Главный экран. Показывает остаток калорий на сегодня, прогресс-бар,
│     лог съеденного за день с временем, грид пресетов по категориям.
│     Тап на пресет → логирует калории с анимацией и toast-undo.
│     При 0 пресетах показывает кнопку перехода на presets.html.
│
├── presets.html
│     Управление пресетами. Фильтры по категориям (завтрак/обед/ужин/перекус/напитки).
│     Хинт для каждой категории что туда добавлять. Тап на пресет раскрывает
│     кнопки "Редактировать" и "Удалить". Модалка добавления/редактирования
│     с выбором фото (base64) или эмодзи из грида.
│
├── history.html
│     История приёмов пищи. Три фильтра: дни / недели / месяцы.
│     Каждая строчка: дата, съедено ккал, отклонение от нормы (+/-) цветом.
│     Тап раскрывает детали дня со списком записей и удалением по тапу.
│     Сводка вверху: streak, дней в норме, дней с перебором.
│
├── settings.html
│     Настройки. Изменить дневную цель калорий (вручную или пересчитать).
│     Изменить цель (похудеть/держать/набрать). Danger zone: очистить историю
│     или сбросить всё (удаляет localStorage и возвращает на онбординг).
│
│   # ── PWA файлы ────────────────────────────────────────────────────────────
│
├── manifest.json
│     PWA манифест. Название приложения, цвета (theme/background), ссылки
│     на иконки. Нужен чтобы браузер предложил "Добавить на экран домой"
│     и приложение открывалось без адресной строки (display: standalone).
│
├── sw.js
│     Service Worker. При установке кэширует все HTML/JS/иконки.
│     Стратегия cache-first: отдаёт из кэша, при отсутствии идёт в сеть.
│     Обеспечивает офлайн работу после первого открытия.
│     При обновлении (новый CACHE key) удаляет старый кэш.
│
├── icon-192.png
│     Иконка приложения 192×192px. Используется Android и manifest.
│     Заменить на финальный дизайн перед релизом.
│
├── icon-512.png
│     Иконка приложения 512×512px. Используется для splash screen и
│     Google Play (если будет нативная обёртка). Заменить перед релизом.
│
│   # ── Assets (пока не используется, для V2) ──────────────────────────────
│
├── assets/
│   ├── css/
│   │   └── app.css     V2: общие стили когда Tailwind будет локальной сборкой
│   └── js/
│       └── app.js      V2: общий JS (утилиты, константы) вынесенный из HTML
│
│   # ── Сервер ───────────────────────────────────────────────────────────────
│
└── .htaccess
      Apache конфиг. Редирект HTTP → HTTPS (обязателен: service worker
      работает только на HTTPS). Без этого файла PWA не установится на iPhone.


LOCALSTORAGE SCHEMA
• goal              — строка: 'lose' | 'keep' | 'gain'
• calories          — число: дневная цель калорий (например 2000)
• presets           — JSON массив: [{name, kcal, cat, emoji, photo}]
• logs_YYYY-MM-DD   — JSON массив за каждый день: [{name, kcal, emoji, photo, time}]


INSTALL AS PWA (один раз)
• Задеплоить файлы на хостинг с HTTPS
• iPhone: открыть в Safari → Поделиться → «На экран Домой»
• Android: открыть в Chrome → меню ⋮ → «Добавить на главный экран»


NON-OBVIOUS DECISIONS
• Нет бэкенда намеренно — MVP проверяет поведенческую гипотезу (будет ли
  юзер логировать каждый день) без инфраструктурных затрат. Бэк добавляется
  в V2 только если retention подтверждён.
• Tailwind через CDN — для MVP достаточно, в V2 заменить на локальную сборку
  для офлайн работы без интернета при первом запуске.
• Фото пресетов сохраняются как base64 в localStorage — ресайзятся до 400px
  чтобы не переполнять хранилище (лимит ~5MB на домен).
• JS-селекторы через data-js="name" вместо id/class — чтобы не смешивать
  логику и стили, легче искать по проекту.
• pageshow + visibilitychange в index.html — фикс bfcache: при возврате
  со страницы пресетов данные перечитываются из localStorage.
• service worker использует cache-first стратегию — приложение работает
  офлайн после первого открытия.
